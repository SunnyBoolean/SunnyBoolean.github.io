Opencv环境搭建

发表于 2018-07-13 | 分类于 OpenCv Android之旅

       本系列开始学习Opencv。因为之前业务需要研究过Opencv，但是苦于网上的Opencv Android相关知识比较少，且很零碎，所以我觉得有必要对Opencv Fro Android进行一个学习总结并形成博文。

OpenCv的三种使用方式

       对于Opencv的介绍我就不多说了，其功能就是图像处理。在Android里使用Opencv有三种方式：

  1. 使用OpenCv Manager+ OpenCv android sdk
  2. 使用OpenCv android sdk + OpenCv动态库
  3. 使用OpenCv动态库+JNI。

       第一种方式对于开发者来说是最简单的，但是对用户是最不友好的，因为要想使用OpenCv的功能还需要安装OpenCv Manager这个APK，试想一下如果用户要用你的app还得去安装别的app谁能接受呢？当然了，使用这种方式对于开发者来说是非常快速的，因为OpenCv提供的AndroidSDK提供了一套Java层的接口，因此开发人员只需要跟java打交道，不需要接触jni。

       使用第二种方式即Opencv android sdk+动态库，可以避免安装OpenCv Manager app但是需要开发人员利用make或cmake自行编译动态库，同时会使我们的app体积增大。而且有的功能opencv sdk并没有提供，依然需要些jni自己实现。

       使用第三种方式就是直接使用Opencv动态库+JNI的方式进行开发，这种方式要求开发人员掌握NDK开发、C++语言。这种方式性能高、可拓展性强、拥有很大自由度。虽然开发难度比较大，但是相比较于优点我个人还是倾向于选用此种方式来使用Opencv。

OpenCv在AS中的环境搭建

       对于我个人来讲，我选择第三种方式，即Opencv动态库+JNI来使用Opencv，所以，本篇讲的就是以这种方式来搭建环境。

下载OpenCv Android SDK

       下载地址：https://opencv.org/releases.html，下载后解压即可。

创建App项目

       创建app时需要勾选C++支持，具体的请看NDK开发系列的《NDK开发环境搭建》。

       项目创建app后把下载解压后的Opencv sdk的sdk/native/jni目录下的include目录整个都拷贝到main/cpp目录下。同时把sdk/native/libs下相关的libopencv_java3.so文件拷贝到src/main/jniLibs目录下，如果app里没有这个目录，就手动创建一下。
